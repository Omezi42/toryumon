# 「触るだけで楽しい」タイトル画面＆UI/UXインタラクション設計書

スマホアプリや高品質なインディーズゲームの「気持ちよさ」は、静止画の美しさではなく、「ボタンを押したときの絶妙な感触」「画面が動いたときの心地よいアニメーション」といった、徹底的なUI/UXのこだわりから生まれます。

画像アセットに一切頼らず、Godotの機能（Tween、Shader、Draw、Particles）を極限まで使い倒し、タイトル画面を開いただけでプレイヤーを虜にするJUICYな設計を定義します。

---

## 1. タイトル画面の「奥行き」と動的ビジュアル（JUICYビジュアル）

ただの1枚絵のタイトル画面は、どれだけ美麗でも退屈です。画面を多層構造（レイヤー）にし、異なる速度で動かすことで、吸い込まれるような奥行きを生み出します。

### 【タイトル画面の多層レイヤー構造】
```
 [レイヤー 5: 最手前] ───────＞ 煌めく水滴・霧のパーティクル（カメラのすぐ前を通り抜ける）
 [レイヤー 4: UI/ロゴ] ──────＞ 筆文字ロゴ、和風モダンな極細ボタン群（触ると弾む）
 [レイヤー 3: インタラクティブ] ＞ 「緋鯉」（タッチした場所へなめらかに泳ぎ、波紋を描く）
 [レイヤー 2: 背景（崖）] ───＞ 左右のゴツゴツした崖（Draw関数描画、少しだけカメラワークで揺れる）
 [レイヤー 1: 最背面（滝）] ─＞ 滝シェーダー/描画（上から下へ、高速で美しく流れる）
```

### ① 鯉（コイノスケ）のインタラクティブ・スイム
- **挙動**：タイトル画面の中央で、白と朱の美しい鯉（SVGアセット/Draw描画）が退屈そうにゆらゆらと泳いでいます。
- **インタラクション**：
  プレイヤーが画面のどこかをタップ（またはマウスホバー/クリック）すると、その場所に向けて鯉が滑らかなカーブを描いて泳いでいきます。
  タップした瞬間、その場所を中心に美しい「水紋（リング状の白い波紋）」がスッと広がり、消えていきます（Godotの `_draw()` で描画）。

### ② ジャイロ / マウス連動の「パララックス（視差）」
- **効果**：スマホを少し傾ける（またはPCでマウスを動かす）と、手前の「崖」と、背後の「滝」、そして最手前の「水しぶきパーティクル」が、それぞれ異なる比率でごく僅かにズレて動きます（パララックス効果）。
- **プレイヤーの心理**：これだけで「画面の中に生きた空間がある」と感じられ、意味もなくスマホを傾けたりマウスを振ったりしたくなります。

## 2. 「触り心地」を極めるマイクロインタラクション

「ボタンを押す」という行為を、最高のエンターテインメントに変えます。

### ① ボタンの「Squash & Stretch（潰れと伸び）」
- **ホバー時**：ボタンがほんの少しだけ「ぷっくり」と膨らみ、左右に小さく揺れます（1.08倍にスケールアップ）。
- **プレス時**：指でグッと押し込まれたように、縦に潰れて横に伸びます（スケール `Vector2(1.15, 0.85)`）。
- **リリース（決定）時**：弾むように元の形に戻りながら、ボタンの背景から「白練色（白）の水しぶき」がファサッと弾け飛びます。

### ② 水滴タップSEと和太鼓の音響設計
- **ホバー音**：竹が風に揺れるような繊細で心地よい音、あるいは澄んだ「ピチョン」という水滴音。
- **決定音**：画面全体が引き締まる、上品で響き渡る「和太鼓の ── ッドンッ！」という深い低音。

## 3. 画面遷移の美学：「滝登りフェード」

「プレイ開始」を押した瞬間のトランジションがゲームの期待値を最高潮に高めます。

### JUICY（滝登りフェード）：
1. 「開始」ボタンを押した瞬間、すべてのUIボタンが画面の左右に「サッ」とスライドして退場。
2. 同時に、BGMの滝の音が「ザザザザーッ！」と激しさを増す。
3. 画面最下部から、激しい白い泡（水しぶき）の波頭が、画面の上に向かって一気にスクロールして画面全体を覆い尽くす。
4. その泡が通り抜けたときには、タイトル画面からシームレスに次のプレイ画面へと切り替わっている。

## 4. 本物のアプリ感を引き出すUIディテール設定

| UI要素 | 設定箇所 | 設定値 / 調整方法 | 効果 |
| :--- | :--- | :--- | :--- |
| **タイトルロゴの配置** | Control -> Theme Override | Noto Serif JP風 (明朝体) / 54〜72px / 太字 | 静寂の中に格式高い和の高級感を与える。 |
| **ボタンの枠線** | StyleBoxFlat -> Border | Width: 2px / Corner Radius: 4px | 線の太さを細く保ち、現代アートに近い清潔感を出す。 |
| **UIの余白（マージン）** | MarginContainer -> Theme Overrides | Left/Right: 64px, Top/Bottom: 48px | 画面の端から十分な「余白の美」を持たせ、画面を窮屈にさせない。 |
| **パララックス感度** | Node2D / Control スクリプト制御 | 崖 `pos += マウス * 0.02` / 最背面 `pos += マウス * 0.005` | 視差効果が「強すぎない（酔わない）」心地よい奥行きを再現。 |
