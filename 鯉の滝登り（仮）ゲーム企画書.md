# **『鯉の滝登り（仮）〜登竜門アクション〜』ゲーム企画書**

本書は、Godot Engineを用いて開発し、unityroomにて完全無料で公開・運営するためのステージクリア型アクションゲーム of 企画書です。Firebaseを活用したステージ共有機能や、SNS（X）でバズを生み出すためのリプレイ・動画保存システムを取り入れ、個人でも短期間かつ高品質に制作できる設計としています。

| 項目 | 内容 |
| :---- | :---- |
| **タイトル（仮）** | **登竜門！爆走コイキング（仮）** または **龍門の軌跡 〜鯉が龍になる日〜** |
| **ジャンル** | 縦スクロール・レーンアクションゲーム（ステージクリア型 ＋ エディター要素） |

## **2\. 企画の「弱い部分」の補強案（ゲーム性の向上）**

自動で登るレーンアクションはシンプルで分かりやすい反面、単調になりがちです。以下のシステムを導入することで、プレイヤーに「駆け引き」と「爽快感」を提供します。

### **① ダッシュ（DASH）の「リスクとリターン」の再設計**

* **補強案：『スタミナ（呼吸）システム』**  
  * 画面に「スタミナゲージ」を設置。ダッシュ中は激しくスタミナを消費する。  
  * スタミナが切れると「息切れ状態」になり、一時的に通常より移動速度が遅くなる。  
  * **リターン**：ダッシュ中のみ、特定の「壊せる障害物（流木や小さな氷）」を体当たりで破壊できる。また、後述の「気流（急流）」を逆らうのに必須。

### **② レーンごとの特性（環境変化）**

* 3つのレーンすべてが同じ流れではなく、秒単位で「急流（押し流されるレーン）」や「上昇気流（スピードアップするレーン）」が発生。  
* ただ避けるだけでなく、「どのレーンを通るのが最も速く安全か」を瞬時に判断するパズル的要素を追加。

## **3\. 障害物のバリエーション＆ユニークギミック**

縦スクロールを飽きさせないための、ユニークでユーモアのある障害物プランです。

| 障害物カテゴリ | 名称 | 挙動・特徴 | プレイヤーの対策 |
| :---- | :---- | :---- | :---- |
| **固定** | **岩・巨岩** | 完全にレーンを塞ぐ。動かない。 | 回避必須。 |
| **動物系** | **サルの投石** | 滝の両端（崖）から、一定周期で横方向に岩を転がしてくる。落とす前にアラートマークが表示される。 | タイミングを見てダッシュで通り抜ける。 |
| **動物系** | **クマのサケハント** | 滝の中央 of 岩の上から、横向きに「鮭（または別の魚）」を投げてくる。 | 鮭に当たると横レーンに弾かれる。 |
| **ユニーク** | **釣り人の釣り針** | 画面外上部から釣り糸が垂れてくる。ゆらゆらとレーンを左右に動く。 | 針に引っかかると数秒間スクロールが停止し、下流へ引きずり戻される。 |
| **ユニーク** | **UFOのキャトルミューティレーション** | 突如上空にUFOが出現。1つのレーンにビームを照射し、吸い上げ範囲を形成。 | ビームに触れると、別のレーンに強制テレポートさせられる。 |

## **4\. 全10ステージ構成案（1ステージ約30秒の高速設計）**

各ステージは**1プレイ30秒以内**で決着がつく、テンポが良く中毒性の高いハイスピード設計とします。少しのミスが命取りになるシビアさと、ダッシュを駆使して最速で駆け抜ける爽快感を両立させ、タイムアタックの面白さを引き出します。

| Stage | ステージ名 | 主要ギミック | 目標タイム（秒） | 概要と設計意図 |
| :---- | :---- | :---- | :---- | :---- |
| **1** | **始まりの滝** | 固定の岩 | 15〜20秒 | 操作方法（レーン移動、ダッシュ）を学ぶチュートリアル。 |
| **2** | **逆風と急流** | 環境レーン変化 | 20〜25秒 | 「急流（減速）」と「上昇気流（加速）」を初導入。最速ルート選択を学ぶ。 |
| **3** | **落石注意の崖** | サルの投石 | 25〜30秒 | 横から岩が転がってくる。ダッシュの無敵（または突破力）の使いどころ。 |
| **4** | **サケ飛び交う急流** | クマのサケハント | 25〜30秒 | 横から鮭が飛んでくる。当たると強制的に別レーンへ弾かれるスピード感。 |
| **5** | **釣り人の縄張り** | 釣り針 | 25〜30秒 | ゆらゆら動く「釣り針」を初導入。焦ると引っかかるパズル的配置。 |
| **6** | **未知との遭遇** | UFOの光線 | 25〜30秒 | 「UFOの強制テレポート」が初登場。瞬時のレーン切り替えを求める脳トレ。 |
| **7** | **サルの悪巧み** | 投石 ＋ 急流 | 30秒 | 投石を避けつつ、急流に逆らうためにダッシュスタミナを管理する。 |
| **8** | **クマと釣り針の迷宮** | サケ ＋ 釣り針 | 30秒 | 鮭に弾かれて横に移動した先に釣り針が待つ、緻密な操作が求められる。 |
| **9** | **カオスの急流滝** | 全ギミック複合 | 30秒 | これまでのギミックがほぼすべて登場。エディターで見たくなる賑やかさ。 |
| **10** | **登竜門** | 全ギミック ＋ ラストスパート | 30〜35秒 | 最難関。駆け抜けてゴールした瞬間に「龍」へ進化し、ステージエディターが解放！ |

## **5\. Firebaseによる完全無料の「ステージ共有システム」**

サーバー維持費を完全無料に抑えつつ、ユーザーが作ったステージをゲーム内でシームレスに共有するため、**Firebase** の無料枠（Sparkプラン）を最大活用します。

### **① 使用するFirebaseサービス**

* **Firebase Authentication (匿名認証)**  
  * ユーザーに面倒な登録をさせず、バックグラウンドで「匿名アカウント」を自動作成。これにより、ステージの投稿者識別や不正防止をセキュアに行います。  
* **Cloud Firestore (NoSQLデータベース)**  
  * ステージデータ（岩や障害物の座標、ステージ名、作成者ID、クリアタイムなど）を保存。  
  * **無料枠**：読み取り 50,000回/日、書き込み 20,000回/日。個人開発・unityroom公開レベルであれば、この無料枠内で完全に賄うことができます。

### **② 無料枠を使い切らないための最適化手法**

* **データの軽量化**：ステージデータをJSONや圧縮文字列にし、1つのドキュメントのサイズを極小（数KB以下）に抑えます。  
* **読み込み回数の節約（キャッシュ・制限）**：  
  * ゲーム起動時に一気に読み込まず、「人気順」「新着順」などのリストを1回につき最大20件程度に制限（ページネーションの実装）。  
  * プレイ済みのステージ情報はローカル（IndexedDB）にキャッシュし、Firestoreへの無駄なリクエストを削減します。

## **6\. リプレイ再生 ＆ 動画書き出し（X共有）システム**

ゲームの面白さを最も伝えやすいのが「動画」です。X（旧Twitter）上で「この超難関ステージをクリアした！」「こんな面白いリプレイがある！」と拡散されるための、技術的な実現アプローチです。

### **① 軽量「入力ログ（Input Log）」式リプレイ機能**

* **課題**：リプレイを「動画ファイル」としてデータベースに保存すると、容量がすぐに数GBに達し、Firebaseの無料枠を使い切ってしまいます。  
* **解決策**：リプレイを「プレイヤーの入力データ」または「座標ログ」として保存します。  
  * プレイヤーが「何フレーム目にどのレーンへ移動したか」「いつダッシュしたか」という軽量なテキストデータ（インプットログ）を保存します。  
  * 再生時は、ゲームがそのログを再現してプレイヤーを自動操作します。  
  * **メリット**：データサイズが数KBで済むため、Firebase Firestoreにステージデータと一緒に保存でき、完全に無料で他人のリプレイをゲーム内で再生可能です。

### **② 「ステージ開始〜ゴール」完全録画システム（WebM書き出し）**

プレイ中の動画を「ステージの開始からゴール（またはゲームオーバー）まで」きっかり切り取って保存できるようにします。

* **技術的仕組み（Godot 4.x × JavaScript連携）**：  
  * unityroom（Webブラウザ）上で動作する利点を活かし、ブラウザの **MediaRecorder API** を使用します。  
  * Godot内のJavaScriptブリッジ機能（JavaScriptBridge）を使い、HTMLの \<canvas\> 要素（ゲーム画面）の描画ストリームを直接キャプチャして録画します。  
* **確実な期間切り出しのワークフロー**：  
  1. **【開始】**：プレイヤーがステージを開始（またはカウントダウンが終了して動けるようになった瞬間）に、GodotからJavaScriptを呼び出し、MediaRecorder.start() で録画を開始します。  
  2. **【ゲーム中】**：ブラウザ側のメモリバッファにゲーム画面の描画（ストリーム）が蓄積されます。  
  3. **【終了】**：プレイヤーがゴールに到達した瞬間（またはゲームオーバーになった瞬間）に、即座に MediaRecorder.stop() を呼び出し、録画を停止します。  
  4. **【書き出し】**：録画データをWebM形式の動画ファイルに変換し、ユーザーの端末（PC/スマホ）に自動ダウンロードさせます。  
* **メリット**：  
  * 「きっかり開始から終わりまで」が収録されるため、無駄な余白やロード画面が入らず、そのままSNSに投稿できる高クオリティなゲームプレイ動画になります。  
  * サーバー不要（完全無料）で動作します。

### **③ X共有のエンゲージメント設計**

* シェアボタンを押した際、あらかじめ以下のような投稿テキストを自動生成します。🌊 激ムズ滝登りステージをクリア！ 🐉  
  ステージ名：『サルの大暴れ滝』  
  クリアタイム：00:42.15  
  👇このコードを入力して挑戦しよう！  
  【ステージコード： \#123456 】  
  \#コイの滝登り \#unityroom \#GodotEngine \[ゲームのURL\]  
* これにより、テキスト、動画（ローカルから手動添付）、そしてプレイするための動線（ステージコードやURL）がすべて揃った最強の拡散フォーマットが完成します。
